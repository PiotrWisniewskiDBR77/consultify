# Documentation Overview

This repository follows an enterprise‑grade documentation hierarchy. The top‑level `docs/` directory contains the following sections:

- `00_README.md` – this guide and navigation map.
- `01_Product/` – product vision, positioning, principles, and lifecycle (phases A‑F).
- `02_Business/` – business model, pricing & trial logic, referral & partner model.
- `03_Users_and_Roles/` – user types, roles & permissions, consultant mode.
- `04_Functional_Specs/` – detailed functional specifications for each phase (A‑F).
- `04_Functional_Specs/Phase_A_Landing.md` – Phase A Landing functional specification
- `04_Functional_Specs/Phase_B_Demo.md` – Phase B Demo functional specification
- `05_AI_System/` – AI role, onboarding, decision support, autonomy & guardrails.
- `06_Architecture/` – system architecture, API integrations, data model.
- `07_UI_UX/` – UX principles and key flows.
- `08_Security_and_Compliance/` – security principles and compliance standards.
- `09_Delivery_and_Governance/` – delivery model and roadmap.
- `archive/` – legacy or non‑canonical documents (e.g., old coverage plans, Word docs).

> **"This repository is the single source of truth for Consultinity."**

Navigate to the appropriate sub‑folder to find the authoritative documentation for a given topic.
