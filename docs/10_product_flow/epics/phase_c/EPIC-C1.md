# EPIC-C1: Access as a Commitment Signal

| Field | Value |
|-------|-------|
| **Epic ID** | EPIC-C1 |
| **Phase** | C — Trial Entry |
| **User State** | `TRIAL_TRUSTED` |
| **Priority** | CRITICAL |

---

## Intent

Access **filters seriousness**.

---

## System Requirements

### MUST
- [ ] Require **invitation or referral**
- [ ] Explain **why access is limited**

### MUST NOT
- [ ] Provide instant gratification
- [ ] Allow anonymous trial

---

## Acceptance Criteria

| Criteria | Pass Condition |
|----------|----------------|
| Trial feels **selective and intentional** | ✅ |
| User understands **why** access was granted | ✅ |

---

## Access Mechanisms

| Type | Description |
|------|-------------|
| Invitation Code | Direct code from existing user |
| Referral | Link from trusted source |
| Consultant Invite | Professional introduction |
| Team Invite | Organizational invitation |
