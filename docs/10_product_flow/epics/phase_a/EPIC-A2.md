# EPIC-A2: Trust Built Through Restraint

| Field | Value |
|-------|-------|
| **Epic ID** | EPIC-A2 |
| **Phase** | A — Pre-Entry |
| **User State** | `ANON` |
| **Priority** | HIGH |
| **Non-Negotiable** | YES |

---

## Intent

Trust is built by **what the system refuses to do**, not what it shows.

This epic ensures the system demonstrates maturity by **deliberate limitation**.

---

## User Context

The user is testing whether this system **respects their intelligence**.

---

## System Requirements

### MUST
- [ ] Present exactly **one CTA**
- [ ] Keep communication **sparse and calm**
- [ ] Allow disengagement without consequence

### MUST NOT
- [ ] Create urgency
- [ ] Include social proof
- [ ] Optimize for clicks

---

## Acceptance Criteria

| Criteria | Pass Condition |
|----------|----------------|
| CTA reads as **invitation**, not funnel | ✅ |
| User feels **no pressure** to continue | ✅ |

---

## Failure Modes

| Mode | Description |
|------|-------------|
| ❌ Multiple CTAs | More than one call-to-action |
| ❌ Persuasive Microcopy | Sales-driven language |
| ❌ FOMO Triggers | Urgency or scarcity tactics |
